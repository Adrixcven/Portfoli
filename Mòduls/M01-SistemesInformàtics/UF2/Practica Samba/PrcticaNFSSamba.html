Pràctica NFS/Samba 

En una empresa volem donar accés als usuaris a certes carpetes d’un servidor amb Debian Linux.

L’empresa té els departaments de direcció, compres, vendes i desenvolupament.

Al departament de desenvolupament treballen tres persones: el Joan, el Toni i la Cristina.

Al departament de compres només hi ha el Pau, a vendes la Roser i el Jordi i a direcció el Marcel (que és el director) i la Carme.

Cada departament es troba en una subxarxa com s’indica a continuació:

  • Desevolupament: 192.168.1.0/24
  • Compres: 192.168.2.0/24
  • Vendes: 192.168.3.0/24
  • Direcció: 192.168.4.0/24

Para empezar el trabajo, crearemos los usuarios que nos pida con el comando “sudo useradd -s /bin/false nombre”, poniendo /bin/false para que no puedan iniciar sesion en el servidor.

Después, añadiremos estos usuarios a samba con “sudo smbpasswd -a usuario”

A continuación, crearemos los 4 grupos para los usuarios

Y, por último, añadiremos los usuarios a sus respectivos grupos.

1.- Instal·la el necessari per tenir el servei NFS a la màquina Debian.  Indica com reinicies el servei NFS i mostra que s’està executant (mostra que el servei escolta en el port corresponent i que el seu estat és actiu). Quin port utilitza el servei NFS?

Instalamos el servicio NFS con el comando “sudo apt install nfs-kernel-server”.

Para reiniciar el servicio, haremos el comando “sudo systemctl restart nfs–kernel-server.service” y miraremos que esté funcionando con “sudo systemctl status nfs–kernel-server.service”.

El puerto que usa el servicio NFS es el puerto 2049

2.- Configura el servidor per permetre que els usuaris del departament de desenvolupament puguin accedir per NFS a la carpeta devel del servidor en mode lectura/escriptura i la resta de departaments només en mode lectura.  Quin canvi faries a la configuració del servidor NFS per tal de permetre que un departament pugui escriure?

Para configurar la carpeta, entraremos en el archivo /etc/exports y pondremos la carpeta y las redes: “ /devel 192.168.1.0/24(rw,async,no_subtree_check)

/devel 192.168.2.0/24(ro,async,no_subtree_check),

/devel 192.168.3.0/24(ro,async,no_subtree_check),

/devel 192.168.4.0/24(ro,async,no_subtree_check)”

Una vez las tengamos, reiniciamos o recargamos el servicio con el comando “sudo systemctl restart/reload nfs-kernel-server.service” y con “sudo systemctl status nfs-kernel-server.service” nos aseguraremos de que no haya ningún error

El cambio que haríamos para que los demás departamentos puedan escribir en el directorio sería cambiar el “ro” del /etc/exports por un “rw”.

Detalla la configuració del servidor i els clients que necessites perquè es munti automàticament la carpeta en xarxa per NFS.

Para que se pueda montar automáticamente en el cliente la carpeta en red, primero en el servidor deberemos verificar que esta subida con el comando “sudo exportfs”

Una vez verificado, entraremos en el cliente e instalaremos el nfs-common con el comando “sudo apt install nfs-common”

Cuando lo tengamos instalado, entraremos en el archivo /etc/fstab y pondremos: “192.168.1.10:/devel /desarrollador nfs rw 0 0”, siendo rw para el departamento de desarrollo y ro para los demás departamentos.

Ahora, se montará automáticamente en cuanto tengamos el servidor y el cliente iniciados. Aunque en esta sesión no se montara a menos que pongamos el comando “sudo mount -t nfs 192.168.1.10:/devel /desarrollador”

Si montamos con la ip que puede escribir, no nos dará ningún error al editar ni crear nada.

Si montamos con la ip que no pueda escribir, al intentar crear algo dará error.

3.- La Cristina utilitza també una màquina Windows 10 per accedir a la carpeta devel en mode lectura/escriptura.

Instal·la el servei Samba al servidor per tal de permetre que pugui accedir.

Para instalar Samba en el servidor, deberemos hacer el comando “sudo apt install samba”

Fes la configuració del servidor suposant que potser més endavant, els altres usuaris del departament de desenvolupament també hauran de poder accedir amb els mateixos permisos que tenen amb NFS.

Detalla la configuració del servidor.

Para añadir en la configuración del servidor del Samba la carpeta, entraremos en el archivo “/etc/samba/smb.config” (haciendo una copia de seguridad antes) y, dentro de ella, haremos una nueva sección llamada “devel” donde pondremos las opciones:

“path = /devel” para seleccionar la carpeta

“writable = yes” para añadir que la carpeta sea lectura y escritura

“valid list = @desenvolupament” para poner que solo los usuarios del grupo desenvolupament puedan acceder

Para probar que la configuración es correcta, haremos el comando “testparm” y este nos dirá si el archivo está bien o no.

Para acceder a la carpeta desde Windows, simplemente deberemos entrar en el explorador de archivos, haciendo clic en “Redes”, nos conectaremos a una unidad de red. Esta nos pedirá la dirección del recurso compartido y el usuario con el que entraremos.

Mostra com veuria un client del departament de desenvolupament la seva màquina Windows amb la carpeta devel accesible des de la seva màquina.

Una vez dentro de la carpeta compartida, podremos ver que nos pone la letra a la que se ha conectado, la ip de la que se ha conectado y el nombre de la carpeta.

4.- El servidor Samba només pot deixar accedir a màquines de les xarxes locals on estan els departaments:

  • Desevolupament: 192.168.1.0/24
  • Compres: 192.168.2.0/24
  • Vendes: 192.168.3.0/24
  • Direcció: 192.168.4.0/24

Para hacer que el servidor Samba solo deje acceder a las máquinas de las redes locales, deberemos entrar en el archivo /etc/samba/smb.conf y, en el apartado [global], añadiremos la línea “hosts allow = ‘ip dep1’ ‘ip dep2’ ‘ip dep3’ ‘ip dep4’”.

A més, el servidor ha de poder ser trobat a la xarxa local amb el nom: ServidorDocs.

Para poner que la red local tenga el nombre de ServidorDocs, deberemos entrar en el archivo “/etc/samba/smb.conf” y, en el apartado [global] añadir el apartado “netbios name = ServidorDocs”. Una vez guardado, reiniciamos el servicio “nmbd” y, al entrar desde windows, podremos conectar simplemente con el nombre.

Indica la configuració que has de fer i quins serveis has de reiniciar per fer-la efectiva.

Para hacer las 2 partes, hemos tenido que reiniciar los servicios “smbd” y “nmbd”.

5.- Mostra la configuració del servidor Samba per afegir la resta d’usuaris:

Para hacer este ejercicio, lo primero que haremos será crear las carpetas con el comando “mkdir”

  • La carpeta compres haurà de ser accessible només per al departament de compres i en mode lectura/escriptura.

Para poner la carpeta compres, deberemos crear una nueva sección llamada compres que tenga el path “/compres”, sea de lectura/escritura con “writable = yes” y tenga los usuarios válidos del grupo de compras con “valid users = @compres”. También pondremos que solo puedan acceder los usuarios desde la red del departamento.

Al comprobarlo vemos que, si cambiamos las ips a 192.168.2.x podemos acceder

  • La carpeta vendes haurà de ser accessible en mode lectura/escriptura només per al departament de Vendes, excepte per al Jordi que només tindrà permís de lectura.

Para poner la carpeta vendes, deberemos crear una nueva sección llamada compres que tenga el path “/vendes”, sea de lectura/escritura con “writable = yes”, que tenga los usuarios válidos del grupo de vendes con “valid users = @vendes” y, que por último, tenga a jordi en la lista de solo lectura con “read list = jordi”.

También pondremos que solo puedan acceder los usuarios desde la red del departamento vendes

Al comprobarlo vemos que, si cambiamos las ips a 192.168.3.x podemos acceder y, si entramos con jordi no podremos editar

  • La carpeta direccio haurà de ser accessible tans sols per als membres del departament Direcció i en mode només lectura. El Marcel, de forma excepcional, serà l’únic que hi podrà escriure

Para que la carpeta direccio sea accesible desde windows, añadiremos una sección para direccio que tenga el path de /direccio, que solo puedan entrar los usuarios del grupo direccio,  poner que solo pueda escribir el usuario Marcel y, por último, poner que solo se puedan conectar los que estén en la subred.

Al comprobarlo vemos que, si cambiamos las ips a 192.168.4.x podemos acceder y, si no somos el usuario Marcel, no podremos escribir.

Nota 2: Mostra tant la configuració del servei com els usuaris/grups creats.

6.- Tenim una impressora HP LaserJet 5200 PS sense tarja de xarxa però volem connectar-la a la xarxa perquè es pugui imprimir des de qualsevol departament.

Indica quina configuració faries al servidor i mostra com es veuria des d’un client Windows un cop configurada a la màquina client.

Para conectar la impresora, lo primero que haremos será instalar el servicio cups con el comando “sudo apt install cups”

Una vez instalado el servicio, entraremos en un navegador y entraremos en “localhost:631” (siendo 631 el puerto de CUPS).

Dentro de la web de CUPS, nos dirigiremos al apartado “Administración” y, en él, seleccionaremos “Añadir impresora”. Al hacer esto nos pedirá el usuario y contraseña de un usuario que esté en el grupo lpadmin. Si no tenemos ninguno, lo añadiremos.

Cuando entremos, nos preguntará que tipo de impresora queremos, a lo que elegiremos “AppSocket/HP JetDirect”.

Después de elegir que tipo de impresora, pondremos la conexión a la que se conectara al servidor. Al no existir, con poner una en nuestra subred nos sirve

Una vez puesto, nos pedirá ponerle el nombre, descripción y ubicación. Aqui solo es obligatorio ponerle el nombre

A continuación, nos pedirá la marca para instalar el controlador. Pondremos el que tengamos en el nombre de la impresora, en este caso HP.

Cuando tengamos puesta la marca, seleccionaremos el modelo.

Al acabar, podremos ver que si nos dirigimos a “Impresoras”, estará nuestra impresora creada ahí.

Ahora, para conectarla al cliente, primero deberemos conectar el cliente a la máquina y, cuando estemos conectados, veremos que esta la impresora.

Al ver que la tenemos, haremos click para conectarla. Al ser la primera vez que nos conectamos a ella, nos pedirá que instalemos un controlador, al que pondremos el “Generic/Text”.

Al entrar, nos debería aparecer, pero da un error.

Verifica com es veu des del servei CUPS un document imprès a un client.

7.- Indica què ens aporta Samba que no tingui ja NFSv3.

Samba nos aporta una seguridad ya que nos conectamos a los directorios remotos con usuarios y no con la maquina en si.

8.- Quina diferència hi ha entre Samba i CIFS?

Además de que Samba se usa principalmente para compartir a Windows y CIFS para Ubuntu, Samba es un servicio de código abierto.

9.- Quins serveis has utilitzat a Samba? Per a què serveix cadascun?

En el samba hemos usado 2 servicios, el smbd que nos permite compartir las carpetas en red y el nmbd que nos permite cambiar el nombre con el que identifican al servidor los clientes.

10.- Quin port utilitza Samba?

El puerto principal que usa Samba es el 445

11.- Quina eina de Samba ens permet comprovar que la sintaxi del fitxer de configuració sigui la correcta abans de reiniciar el servei?

La herramienta que nos permite comprobar que la sintaxis del archivo de configuración es correcta es el Testparm.
